# 标识符体系 (Identifiers)

Typedown 支持两种标识符形式：**ID** 和 **Content Hash**。

## ID (标识符)

ID 是实体的名称，用于在作用域内引用实体。

### 格式

- 任何不以 `sha256:` 开头的字符串
- 允许的字符：字母、数字、下划线 `_`、连字符 `-`
- 不允许：空格、斜杠 `/`、点 `.`

### 风格

| 风格 | 示例 | 适用场景 |
| :--- | :--- | :------- |
| **简单名称** | `alice` | 小型项目，局部作用域 |
| **Slug 风格** | `user-alice-v1` | 推荐，命名空间清晰，全局唯一 |

### 作用域

ID 的查找遵循词法作用域规则：
1. 当前文件
2. 当前目录 (`config.td`)
3. 父级目录（递归向上）
4. 全局索引

## Content Hash (内容哈希)

Content Hash 是基于实体内容计算的 SHA-256 哈希值，用于内容寻址。

### 格式

```text
sha256:<64位十六进制字符串>
```

### 计算

```text
SHA-256( Trim( YAML_Content ) )
```

### 用途

- **不可变引用**: 锁定特定版本的内容
- **完整性校验**: 验证内容未被篡改
- **历史追踪**: 在 `former` 字段中引用历史版本

## 解析优先级

当发生引用 `[[target]]` 时，解析器按以下顺序处理：

1. **Hash 检查**: 检查是否符合 `sha256:...` 格式。如果是，直接进行内容寻址。
2. **ID 查找**: 在当前作用域链中按名称查找，若未找到则回退到全局索引。

> **设计意图**: 精确（Hash）优于模糊（ID），确保引用的确定性。
